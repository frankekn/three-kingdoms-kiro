# 先搜後寫

寫 code 前先問：有沒有現成的？

## 流程
1. **定義需求**：需要什麼功能？語言/框架限制？
2. **搜尋**：npm/PyPI、GitHub、專案內既有 skills
3. **評估**：功能匹配度、維護狀態、社群、授權、依賴數
4. **決策**：

| 情況 | 行動 |
|------|------|
| 完全匹配，維護良好 | **直接用** |
| 部分匹配，基礎好 | **包一層 wrapper** |
| 多個弱匹配 | **組合 2-3 個小套件** |
| 找不到合適的 | **自己寫**，但參考搜尋結果 |

## 反模式
- ❌ 沒搜就開始寫 utility
- ❌ 為了一個小功能裝巨大套件
- ❌ 包太多層讓原套件失去優勢
